# Zotero Collection Structure Manager Plugin

![Collection Structure Manager logo](icons/icon-128.png)

Zotero Collection Structure Manager 是一个用于管理 Zotero collection / 文件夹结构的独立插件。它面向文献条目很多、文件夹层级很深、经常需要查找和整理 collection 的 Zotero 文库，把快速跳转、结构查看、常用文件夹管理、条目数量检查、新建、移动和合并等操作集中到一个管理器窗口中。

这个插件的重点不是修改文献条目本身，而是帮助你看清、定位和整理 Zotero 左侧文件夹结构。

## 主要功能

### 文件夹结构浏览与定位

插件会以树形结构列出当前文库中的全部 collection，根文件夹和多层子文件夹都可以逐级展开或折叠。左侧列表采用紧凑单行布局，适合在一个屏幕中查看更多文件夹；展开/折叠按钮也做了强化，方便在大量层级中操作。

搜索支持文件夹名称和完整路径。输入关键词后，插件会显示匹配文件夹及其上级路径，并对关键词进行高亮，便于判断该文件夹处在什么层级。对于重名文件夹，界面会提示其完整路径，避免只看名称时误选。

### 快速跳转与常用位置

选中文件夹后，可以通过双击、回车、按钮或右键菜单跳转到 Zotero 左侧对应 collection。插件还支持复制完整路径，适合在整理文库、记录问题或和他人沟通文件夹位置时使用。

常用文件夹可以加入收藏，近期跳转过的文件夹会进入最近访问。对于大型文库，这两个入口可以减少反复展开层级和重复搜索的时间。

### 条目数量与结构检查

每个文件夹行会显示直接条目数，并支持统计包含子文件夹在内的递归条目数。右侧详情区还会显示子文件夹数量、所在文库和完整路径等信息。

插件可以筛选空文件夹。这里的空文件夹指直接条目、递归条目和子文件夹都为 0 的 collection。它也可以导出 Markdown 或 CSV 格式的文件夹结构报告，便于后续检查、备份或清理规划。

### 新建、移动与整理文件夹

插件支持新建根文件夹，也支持在选中文件夹下新建子文件夹。移动文件夹时采用“源文件夹”和“目标文件夹”的明确选择方式，避免因为当前选中状态不清楚而移动到错误位置。

你可以选择一个或多个源文件夹，把它们移动到根目录或另一个目标文件夹下。移动后会记录最近操作，并支持撤销上一次移动。

### 预览式合并文件夹

合并文件夹用于把一个源文件夹的内容整理到另一个目标文件夹中。执行前会先显示合并预览，并要求输入源文件夹名称确认，降低误操作风险。

合并时会把源文件夹中的直接条目加入目标文件夹，把源文件夹的直接子文件夹移动到目标文件夹下，最后删除源文件夹本身。合并不会删除文献条目、附件、笔记或标签。

## 快速使用

安装并重启 Zotero 后，在 `工具` 菜单中点击 `文件夹结构管理器...` 打开窗口。

快捷键：

- macOS：`Shift + Cmd + M`
- Windows/Linux：`Ctrl + Shift + M`

基本流程：

1. 在搜索框输入文件夹名称或路径，快速定位目标文件夹
2. 单击选中文件夹，双击或按回车跳转到 Zotero 对应 collection
3. 右键某一行，可设为源/目标、跳转、复制路径、收藏/取消收藏、展开/折叠
4. 多选文件夹后，可批量收藏、移动，或作为整理操作的源文件夹

## 移动与合并

### 移动文件夹

移动操作适合把多个分散的 collection 重新放到同一个父文件夹下，或把某些文件夹移到根目录。

操作步骤：

1. 在左侧选择一个或多个文件夹
2. 点击 `设为源文件夹`
3. 选择目标文件夹后点击 `设为目标文件夹`，或在目标下拉框中选择目标位置
4. 点击 `移动到目标位置`

目标位置可以是具体文件夹，也可以是根目录。插件会阻止把文件夹移动到自身或自己的子文件夹下。

### 合并文件夹

合并操作适合处理含义相近、重复建立或层级整理后不再需要保留的 collection。

操作步骤：

1. 选择一个源文件夹并点击 `设为源文件夹`
2. 选择一个具体目标文件夹并点击 `设为目标文件夹`
3. 点击 `预览合并`
4. 检查预览内容
5. 输入源文件夹名称确认合并

合并要求目标必须是一个具体文件夹，不能是根目录；源文件夹只能有一个。合并完成后，源文件夹本身会被删除，但其中的文献条目不会被删除。

## 界面与设置

### 窗口布局

管理器窗口可以拖动标题栏移动位置，也可以从四角调整大小，包括右上角。中间分割线可以左右拖动，用于调整左侧文件夹列表和右侧操作面板的宽度。

窗口大小、位置和中间分割线宽度会保存到插件偏好中，下次打开时会尽量恢复到上次的布局。

### 列表显示

左侧文件夹树使用紧凑单行布局，目的是让一屏显示更多 collection。行内会同时显示文件夹名称、递归条目数、直接条目数、收藏状态、最近访问状态和重名提示。

搜索时，匹配文件夹的上级路径会自动展开或显示出来，避免只看到单个命中文件夹而无法判断其所在层级。

### 选择与右键菜单

文件夹选择支持常见的多选方式：单击为单选，`Cmd/Ctrl + 单击` 用于增减选择，`Shift + 单击` 用于连续选择。

左侧文件夹行支持右键菜单。常用操作可以直接在右键菜单中完成，包括设为源文件夹、设为目标文件夹、跳转、复制路径、收藏或取消收藏，以及展开或折叠当前分支。

### 字体与可读性

管理器窗口内可以调整字体大小：

- `Cmd/Ctrl + +`：放大字体
- `Cmd/Ctrl + -`：缩小字体
- `Cmd/Ctrl + 0`：恢复默认字体大小

字体大小设置会保存在本地插件偏好中。

## 安全边界

- 插件不修改文献条目标题、作者、年份、DOI、标签、附件或笔记
- 移动和合并只作用于 Zotero collection 结构
- 所有结构修改操作都会二次确认
- 合并前会显示预览，并要求输入源文件夹名称确认
- 不提供单独删除文件夹功能，避免误删结构
- 收藏和最近访问只保存到插件本地偏好中
- 当前版本已整合 `zotero-collection-quick-jumper-plugin` 的主要快速跳转能力

## 安装

1. 在 Zotero 中打开 `工具 -> 插件`
2. 点击右上角齿轮
3. 选择 `Install Add-on From File...`
4. 选择构建好的 `.xpi`
5. 重启 Zotero

安装文件：

- `dist/zotero-collection-structure-manager-plugin-0.6.3.xpi`

## 构建 XPI

在插件目录运行：

```bash
bash build.sh
```

构建产物会出现在：

- `dist/zotero-collection-structure-manager-plugin-0.6.3.xpi`

## 兼容性

- Zotero 7

当前 manifest 明确标记为兼容 Zotero `7.0.*`。
